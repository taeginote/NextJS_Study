NextJS로 구현

처음 app에 Page.tsx는 기본 페이지다.
그래서 기본적으로 안에 내용을 다 지우고, 


---------------------정적 라우팅-------------------------------------------------------------
처음 app > detail > page.tsx 를 생성
이거는 디테일 페이지다. 

정적 라우팅 => 그냥 위에 처럼 만든다음 local host 3000/detail url로 설정하면 자동을 detail page로 들어감
-------------------------------------------------------------------------------------------

---------------------동적 라우팅-------------------------------------------------------------
Detail page에서 props로 params을 받는다. 
ex)

type Props = {
	params: {
		location: string
	}
}

export default function Detail({ params }: Props) {
	return (
		<>
			<h1>Detail {params.location}</h1>
		</>
	)
}

근데 이것만 바꾼다고 되는것은 아니다. 파일 형식을 변경해야한다.
이전 app > detail > page.tsx
변경 app > [location] page.tsx

이렇게 하니깐 params안에 location안에 인지를 하기때문에 바로바로 동적으로 받을수있다. 
-------------------------------------------------------------------------------------------

---------------------페이지 이동 Link -------------------------------------------------------------
page 이동을 위해서는 Link라는 태그를 사용하자. 
Link는 무조건 href를 넣어서 자신이 갈 url을 작성해준다.
-------------------------------------------------------------------------------------------

---------------------페이지 이동 router -------------------------------------------------------------
import { useRouter } from 'next/navigation'
const router = useRouter()

const GoHome = ()=>{
    router.push('/')
}

return(
    <>
    <button onClick={GoHome}>홈</button>
    </>
)

위에 같이 버튼 이벤트 onClick을 넣게 된다면,
서버 컴포넌트가 아니라 클라이언트 컴포넌트로 변경해야한다. 

그러면 무작정 클라이언트 컴포넌트가 되는걸 막으려면???
이렇게 버튼들을 따로 component로 구분해서 만들면 된다.

그래서 거기에 use client를 넣어주고,
그렇게 공용 컴포넌트로 구분해주면 된다.
