서버컴포넌트
React Server Component를 줄여서 RSC라 부른다.
React18에서 도입
Next에서 사용할때 ServerComponent라 부름

필요한 이유
- 데이터 페칭 (서버에서 데이터를 가져오니깐 빠르다 물리적으로 가까우니깐)
- 보안 
- 캐싱 (랜더링된값을 캐싱하니깐 재사용하니 빠르다)

- JS번들 크기 감소

서버컴포넌트와 기존(클라이언트 컴포넌트 이 두개는 다르다.)

클라이온트는 
use Client라는 키워드를 써야한다. 
하지만 주의사항 클라이언트 컴포넌트에 서버 컴포넌트를 import 할 수 없다.

그래서 클라이언트 컴포넌트에서 서버 컴포넌트를 사용하고 싶을 때는 props로 넘겨줘야 한다.

그러면 그냥 서버컴포넌트만 사용하면 될거같은데 클라이언트 컴포넌트를 사용하는 이유는 무엇인가?

서버컴포넌트에서는 hook과 event를 사용할수없다 그래서 클라이언트 컴포넌트를 사용해야함

그래서 클라이언트 컴포넌트를 트리에 끝으로 보내라는것이다.

그래서 use client라는것을 트리에서 가장 마지막으로 해라 이런 뜻이다.
